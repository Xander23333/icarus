# [arXiv:2605.11467] Drop the Act: Probe-Filtered RL for Faithful Chain-of-Thought Reasoning

- **Authors / Org**: anonymous (pre-print)
- **Published / Updated**: 2026-05-26 (updated)
- **Primary cat / cats**: cs.LG (cross: cs.CL)
- **Link**: https://arxiv.org/abs/2605.11467

## 核心问题
Reasoning models 常在 *内部已经 commit* 答案后再生成 deliberative-looking CoT（"reasoning theater"），既浪费 token、又污染 interpretability。能否在不破坏 RL 训练稳定性的前提下，可靠地识别并惩罚这种 post-commitment 步骤？

## 方法
**ProFIL**：在 frozen base 上一次性训练一个 multi-head attention probe，从 internal activation 检测 post-commitment 步骤；GRPO rollouts 中超过阈值的 step 直接把 advantage 置零。无需人工标注，verifier-derived labels 即可。横跨 GSM8K / LiveCodeBench / ToolUse / MMLU-Redux，模型用 Llama-8B / Qwen-7B。

## 核心结论
- post-commitment theater 减少 11–100%；faithful-fraction 上升；chain length 下降。
- 关键发现：frozen base 上训出的 probe 即使经过 RL 后仍稳定，**没有出现 prior work 预测的 RL-obfuscation 失败模式**（模型不会"学会骗过 probe"）。
- 一个 drop-in GRPO 扩展即可同时拿到 faithfulness ↑ + cost ↓。

## NTP 归类
- **类别**: counter-evidence (对"CoT-as-theater 是 NTP 推理机制级缺陷"的削弱)
- **理由**: 若 CoT theater 是 NTP 范式的机制级病，frozen probe + advantage masking 不应能在不退化任务表现的同时降下来；ProFIL 的实证表明这是 RL objective + readout-level 行为，能被 readout-side 干预修掉。

## 是否提出新理论边界
no — 工程方法 + 实证；但隐含"frozen probe 不被 RL 闭环吃掉"的稳定性命题，是对 *prior 失败预测* 的反例。

## Reviewer note (Xander)
最强反论：probe 训练在 frozen base 上，意味着 ProFIL 假设 "RL 不大幅改 representation"——四个域上似乎确实如此，但若改用更长 RL run 或更强 base shift，probe 可能漂掉。论文没做长期 RL 漂移实验。仍是迄今最直接的 "CoT-theater 可工程修复" 证据，足以削弱该论据线。
