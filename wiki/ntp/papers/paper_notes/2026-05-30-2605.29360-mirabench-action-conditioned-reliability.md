# [arXiv:2605.29360] MiraBench: Evaluating Action-Conditioned Reliability in Robotic World Models

- **Authors / Org**: Tianzhuo Yang et al. (Peking U + collaborators)
- **Published / Updated**: 2026-05-28
- **Primary cat / cats**: cs.AI
- **Link**: https://arxiv.org/abs/2605.29360

## 核心问题
action-conditioned 机器人 world model 越来越被当作 scalable simulator, 但现有评测多看 *视觉保真*; 是否预测的 future 在 *action 命令下* 物理一致、是否对 failure-inducing actions calibrated, 缺独立测量。

## 方法
分层 benchmark, 三级递进: (L1) Physics Adherence (reference-free 物理一致), (L2) Action-Following Fidelity (是否随 task-relevant action 改变), (L3) Optimism Bias Detection (对应 failure 输入是否仍预测 success)。人工标注 16,000+ judgments, 跨任务族/失败类别/leading world model, 评 12 个代表性 model configuration (含 vector-conditioned 与 vision-conditioned)。

## 核心结论
- 所有 12 模型在 L1 上参差, 但 L3 (optimism bias) 普遍显著——failure-inducing action 下仍倾向预测 success。
- L2 (action-following fidelity) 与 visual fidelity 弱相关——\"好看\" ≠ \"听命令\"。
- 三层分层揭示 *action-conditioned reliability* 是与 visual-fidelity 正交的评测轴, 必须单测。

## NTP 归类
- **类别**: NTP-mech (video / world-model 子带的接口层 confound 实证 — 与 C-WM-7 latent-action codebook identifiability 同源)
- **理由**: 给 §10 C-WM-7 (latent-action codebook identifiability deficit) 一条 *正面 evidence-supply* anchor——它没解决 NMI/purity, 但提供了 \"action-following fidelity\" 作 *弱代理 metric*, 在 frontier 不公开 latent purity 的情况下可由第三方独立测量。

## 与已有工作的关系
- 直接补 §10 C-WM-7 (latent-action codebook identifiability deficit) 的 falsifier 三元组之外的 *第四 weak-proxy 通道*: L2 action-following fidelity 不要求 latent 与 ground-truth action NMI ≥ 0.7, 仅要求 model 在不同 action 输入下 output 显著区分。这在 black-box 设定下可测, 把 C-WM-7 \"frontier 不披露 latent codebook\" 的 evidence-supply 缺口部分绕开。
- 与 [2605.30346] YoCausal 同一 tick 出现, 都是 \"video / world model 的 reliability 评测协议\" 范畴, 但分工互补: YoCausal 钉 *因果方向*, MiraBench 钉 *action 命令保真 + optimism bias*。两者 + [2605.30100] Chess-World-Model 三连一起把 \"world model = 视觉好看\" 的 hype 在 *因果 / 状态 / action* 三维度同时拆开。
- 与 §10 C-WM-6 (head-to-head paradigm benchmark deficit) 同向: MiraBench 不是 paradigm-head-to-head, 但提供了 \"未被覆盖的评测维度\" 的具体清单 (L1/L2/L3), 后续若有 NTP-vs-JEPA 公平对照, 这三层 metric 是必填项。
- 与 [2605.26820] (real-world VLA continual) 形成 *评测协议互补*: VLA 子带 continual benchmark + world-model 子带 reliability benchmark, 两侧夹击 \"embodied foundation model 已 deploy\" 的 marketing。

## 是否提出新理论边界
no — 但提供 *可证伪 evidence-supply* 协议: 任何声称 action-conditioned world model 接近 simulator 用途的 frontier model, 必须在 MiraBench L1/L2/L3 三层上同时通过阈值 (优先 L3 optimism bias)。

## Reviewer note (Xander)
最有价值的是 *L3 Optimism Bias Detection* 这一设计——直接戳中 \"video diffusion 倾向于预测 plausible 而非 truthful future\" 这个不可调 prior。最强反驳: \"failure-inducing action\" 本身要靠人工标注, 标注分布偏差会决定 L3 数字; 16k judgments 跨多任务族后人均覆盖度可能不足以校准 frontier-scale model. 即便如此, L3 是 §10 C-WM-7 / C-EMBOD-6 (demo-credibility) 之间最实用的桥——\"demo 看起来 success\" 和 \"模型预测自己 success\" 是两个 evidence-bar, MiraBench L3 把后者操作化了。下一步关键: 把 L3 失败率与 §10 C-REAS-6 (reward-hacking 阶梯回潮率) 的 verifier-of-verifier 失败模式做联读——optimism bias 与 reward hacking 是同一 \"生成模型在不可观测 ground-truth 上 self-confirm\" 现象在不同 pipeline 段的实例化。
