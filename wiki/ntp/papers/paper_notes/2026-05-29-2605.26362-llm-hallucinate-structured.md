# [arXiv:2605.26362] Why LLMs Hallucinate on Structured Knowledge: A Mechanistic Analysis of Reasoning over Linearized Representations

- **Authors / Org**: Shanghao Li, Jinda Han, Yibo Wang, Yuanjie Zhu, Zihe Song, Langzhou He, Kenan Kamel A Alghythee, Philip S. Yu
- **Published / Updated**: 2026-05-25
- **Primary cat / cats**: cs.CL (cs.AI)
- **Link**: https://arxiv.org/abs/2605.26362

## 核心问题
当外部结构化知识 (graph / table) 被 *线性化* 喂进 LLM context 后，即便信息充分，模型仍会幻觉——这是 attention 失焦、FFN ground 失败，还是两者协同？且这种机制能否跨结构 (graph/multi-hop/table) 复用？

## 方法
- 在 *已知 ground-truth 上下文给定* 的 single-hop graph QA 上量化 attention 分布、FFN 表征、最终 logits 三层；
- 用 attention entropy + shortcut-cue concentration 指标定位 attention 失焦；
- 在 FFN 中用 probing + activation patching 看 *提供的知识* 是否被写入 residual；
- 把分析扩展到 multi-hop graph 与 tabular QA 验证可迁移性；
- 由此构造一个 hallucination 检测器，跨三种结构形式 evaluate。

## 核心结论
- **Attention** 把质量集中在 *shortcut-like* 结构 cue（例如 schema token / 关键符号位）而非完整 context，这是 *任务依赖* 的（不同 graph schema 表现差异大）；
- **FFN** 在 hallucination 例上系统性 **fail to ground** 上下文里的事实，模型回退到 parametric memory；
- 跨 single-hop / multi-hop / tabular，*FFN-grounding 失败* 的 signature 是稳定预测因子；*attention concentration* 的预测力更弱、更 task-specific；
- 基于 FFN-grounding 指标的检测器在三种结构上 >0.8 AUC，证实机制是 *通用*、非 schema-specific。

## NTP 归类
- **类别**: NTP-mech （但是 readout/middle-layer 工程层的 mech，不是 paradigm-level）。
- **理由**: 文章把"为什么 in-context knowledge 不被使用"定位到 **FFN-as-grounding-bottleneck**——这是 NTP 训练目标下 FFN 长期作为"参数记忆 store"的副作用，*不靠 prompt/decoder 工程能修* 但 *可被 mid-training auxiliary 修*，因此严格说是 *mech 信号但工程可救*，与 readout-side 主导假设兼容。

## 是否提出新理论边界
no — 但提出 *可证伪机制命题*：
"在 NTP 训练 + linearized-structured context 下，*FFN-grounding 概率* 是 hallucination 的 sufficient statistic；改进 attention sparsity 不会单独提升 grounding。"

## Reviewer note (Xander)
最强部分是 *跨结构稳定性*：FFN-grounding signature 在 graph→multi-hop→table 三种 schema 下保持预测力，否则就只是"在我们 setup 下 attention 失焦"的另一篇 trivia。最弱部分是 *机制 vs corr*：activation patching 只在 single-hop 上做，multi-hop 和 tabular 上仅靠 probe 相关，不能严格断言"修 FFN 即修 hallucination"。在 survey §10 中应作为 *候选六号 confound* 而非新 mech 候选写入；并应把 "FFN-as-grounding-bottleneck" 单列为 N3/N7 后续要追的子线索。
