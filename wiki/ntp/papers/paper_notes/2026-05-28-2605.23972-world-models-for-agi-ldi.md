# [arXiv:2605.23972] Why We Need World Models for AGI: Where LLMs Fail and How World Models May Outperform

- **Authors / Org**: anonymous (pre-print)
- **Published / Updated**: 2026-05-26
- **Primary cat / cats**: cs.AI
- **Link**: https://arxiv.org/abs/2605.23972

## 核心问题
LLM 在 causal reasoning / persistent state tracking / long-horizon planning 上的局限，是否源自 *objective-level* 错配——sequence prediction 不同于 inference over latent environment dynamics？能否用一个干净 testbed 把这个差距分离？

## 方法
提出 **Latent Dynamics Inference (LDI)**：把 language / 多模态观测视为 underlying transition dynamics 的部分证据。构造 **Flux** —— 一个全部以自然语言规则定义的 sequential reasoning 环境；把规则编译成显式 state-transition simulator，对比 (i) 仅在文本观测上跑的 LLM 与 (ii) 在抽取出的 latent state 空间上训的 RL agent。

## 核心结论
- 在 latent state 上训的 RL agent 在 long-horizon 子任务上系统性优于纯文本 LLM。
- 差距随 horizon 增长非线性放大，与 *prompt 长度* 或 *上下文容量* 不一致，作者论证差距来自 representation level (latent dynamics vs surface text)，而非 context budget。
- 提出"text 观测可作 partial evidence，但需要外部机制做 latent dynamics inference"的概念主张。

## NTP 归类
- **类别**: NTP-mech 候选证据（弱版本）
- **理由**: 主张 NTP 的 *objective* 与 latent dynamics inference 在结构上错配，而非 capacity 不足；如果成立，是机制级。但目前 Flux 是 *synthetic, rule-defined* 环境，且把"规则可编译为 simulator"作为前提——这个前提本身偏向 RL 一侧。

## 与已有工作的关系
- 与 C1 (Deterministic Horizon) 正交：C1 谈单 forward pass 容量上界，本篇谈跨 horizon 的 objective 错配。
- 与 `2026-05-27-2605.05438` (semantic-loss anti-collapse) 形成张力：那篇说 "objective 工程层可救"，本篇说 "objective 范式层错配，需换 paradigm"。可同时为真，区别在是否接受"加 latent dynamics head/regularizer"还算 NTP 范式内。
- 与 `2026-05-27-2605.14004` 的 attribute head 形成对照：那篇说局部 NTP + auxiliary head 已能做全局属性；本篇说 long-horizon 任务上 auxiliary head 不够。规模/horizon 阈值是关键变量。

## 是否提出新理论边界
no formal bound — 提出 LDI 概念框架，未给定量分离结果或 impossibility theorem。

## Reviewer note (Xander)
最强反论：把"规则可编译为 simulator"作为对比前提，等于先给 RL agent 装了一个 perfect world model。NTP 一方等价于"必须 *从 text* 推 simulator"的 hard 任务；差距很可能是 *simulator-access 差距* 不是 *objective 范式差距*。论文里 RL agent 不是"另一种 LLM"，而是"另一种 setup"。仍值得纳入，但要明确归类为 *框架级建议* 而非证据。
