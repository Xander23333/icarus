# [arXiv:2605.10799] The Last Word Often Wins: A Format Confound in Chain-of-Thought Corruption Studies

- **Authors / Org**: Gabriel Garcia
- **Published / Updated**: 2026-05-11 / 2026-05-15
- **Primary cat / cats**: cs.LG / cs.LG, cs.AI, cs.CL
- **Link**: https://arxiv.org/abs/2605.10799

## 核心问题
CoT faithfulness 文献广泛使用 "corruption study"（删/改中间步骤看 accuracy 损失）推断"哪个步骤是计算上重要的"。在 GSM8K / MATH 等 benchmark 中，这种推断是否被 **answer line 的位置** 系统性污染？

## 方法
匹配 GSM8K 样本：仅删除末尾的显式 answer 句、保留全部 reasoning，观察 suffix sensitivity 是否塌缩；构造 conflicting-answer prompt（正确推理 + 错误最终答案）；在 3B–7B 五个 open-weight family 上跑；并设计 generation-time vs consumption-time probe。

## 核心结论
- 仅删除末尾答案行，Qwen 2.5-3B 的 suffix sensitivity 塌缩约 19×（N=300, p=0.022）。
- conflicting-answer prompt 在 7B 把 accuracy 拉到接近 0；wrong-answer following 在 3B–7B 强烈，在更大 scale 减弱。
- generation 时早期 commit 答案的比例 < 5%；但 consumption 时模型却严重跟随显式 answer text → corruption 测出的"重要 step"主要是 readout 效应。
- 提出三步 protocol：question-only control + format characterization + all-position sweep。

## NTP 归类
- **类别**: counter-evidence（对 CoT-faithfulness 文献中 "NTP 推理是真计算" 的方法论反驳，同时也削弱"CoT 是表演"的强 NTP-mech 立场）
- **理由**: 现有大量 CoT (un)faithfulness 结论的 effect size 被 format confound 吃掉；任何拿这些结论支撑 NTP-mech / NTP-cap 立场的论述需要重新评估。

## 与已有工作的关系
- 直接挑战 Lanham et al. (Anthropic CoT faithfulness)、Turpin et al. (CoT not always faithful) 等 corruption-style 研究的方法论。
- 与 today 的 Deterministic Horizon 间接相关：若 CoT step 重要性主要是 readout 效应，那么 "depth via CoT" 的有效深度估计需要重新校准。

## 是否提出新理论边界
no — 是方法论 / 实证 correction，不提出新理论 bound。

## Reviewer note (Xander)
单作者、样本 N=300 偏小，但 19× 的塌缩 + conflicting-answer 跨家族复现已经是强信号。这类"清理 baseline"工作往往比新理论更重要——它把整个 NTP-mech 阵营常引的 "CoT 不忠实，所以 LLM 不真推理" 证据链削掉一段。三步 protocol 值得 survey 收录为方法论附录。
