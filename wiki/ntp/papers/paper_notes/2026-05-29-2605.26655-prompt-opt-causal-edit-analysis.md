# [arXiv:2605.26655] Why Prompt Optimization Works, and Why It Sometimes Doesn't: A Causal-Inspired Edit-Level Analysis

- **Authors / Org**: Shuzhi Gong, Hechuan Wen
- **Published / Updated**: 2026-05-26
- **Primary cat / cats**: cs.CL (cs.LG, cs.NE)
- **Link**: https://arxiv.org/abs/2605.26655

## 核心问题
DSpy / TextGrad 等 prompt optimization 在单 benchmark 改进显著，但 *跨任务/跨 backbone 不传递*。失败究竟是 prompt 的什么属性 × 任务的什么属性的交互？

## 方法
- 在多家 prompt 优化框架 × 多个 LLM backbone × 多个 NLP benchmark 上收集优化前后 prompt；
- 用 propensity-adjusted associational analysis（因果 inference 风格 observational study）控制 surface 特征；
- 把 edit 拆为多个 *互补表示*（cognitive-load 标注、surface-text 特征、edit motif）；
- 估计每类 edit family 对每类任务的条件平均效应。

## 核心结论
- "complexity-increasing" 与 "meta-instructional" edits 对数学 / multi-hop reasoning 任务 **负相关**；
- "step-by-step" 与 "meta-cognitive" edits 对 logical / sequential reasoning 任务 **正相关**；
- 结论在三种 edit 表示下一致、跨框架可重现；
- 整体把 prompt-opt 的 *跨任务失败* 解释为 "edit family × task type" 的 *系统性交互*，而非随机噪音。

## NTP 归类
- **类别**: counter-evidence (对 "prompt-level 行为 = 模型机制水平证据" 的弱主张)。
- **理由**: 进一步把 prompt-engineering 效果归因到 *edit 类型对任务 routing 的耦合*——这是 readout/format 层效应而非 mechanism 层；强化方法论原则："prompt-only 改动的证据不能作为 mech 证据使用"。

## 与已有工作的关系
- 与 readout-side 主导假设 (2605.10799/07120/14004/05438/25891/11467) 一脉相承：本文是 *prompt 编辑* 这一额外侧面的 readout 工程化证据。
- 与 [2605.18079] low-precision softmax CoT 的反读类似：CoT 效用受 *表面* (精度/format/edit type) 调制，而非来源于深层 mech。
- 与 N3 §3-§4 (three puzzle pieces, faithfulness) 在 methodological 层补一条：CoT-style prompt edits 的效果跨 task 不传递这一观察，本身是 *faithfulness 的 indirect 证据* (effect 既依赖 surface family 也依赖 task routing)。
- 与 [2605.26362] 今日 hallucination-FFN-grounding 是 *同一方法论原则的两侧*：两文都说"看起来是 reasoning 的事实，机制级原因在 *接口* 而非 *核心计算*"。

## 是否提出新理论边界
no — 但提出可证伪 *方法论命题*: "任何声称 prompt 改动反映 *模型* 能力提升的工作，必须报告 edit 在该 task family 上的 *propensity-adjusted* 效应；否则不可作为 mech evidence。"

## Reviewer note (Xander)
方法学价值大于结论本身：用 propensity-adjusted associational analysis + 多 representation triangulation 是 prompt 研究该有的标准做法。最弱处是 *causal* 标签名不副实——observational 数据上的 confounder 控制再细也不是 do-operator；论文严谨地用 "causal-inspired" 但流传中往往被简化。在 survey §10 招标书的 *五项 confound* 后应追加一条：**Edit-family × Task-routing 交互**——任何声称跨任务 prompt-opt 成功的论证默认 untrusted，除非给出 edit-family conditional effect 报告。
