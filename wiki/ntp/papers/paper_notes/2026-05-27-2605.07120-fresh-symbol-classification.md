# [arXiv:2605.07120] When Symbol Names Should Not Matter: A Logistic Theory of Fresh-Symbol Classification

- **Authors / Org**: 单作者理论工作 (cs.LG)
- **Published / Updated**: 2026-05-08
- **Primary cat / cats**: cs.LG
- **Link**: https://arxiv.org/abs/2605.07120

## 核心问题
Template task 测的"transformer 是否在做抽象符号推理而非记忆 token 名"——在固定标签分类（非 next-token 生成）的弱化版本下，能否给出 closed-form 误差分解，把"真正的 template-level 决策"与"训练集偶然 token 共现"分开？

## 方法
在 transformer-kernel regime 下分析正则化 kernel logistic classification；对训练-测试词汇可不相交的 template 任务，把学到的分类器分解为：(i) ideal template-level classifier (与 token name 无关)，(ii) 来自训练 vocabulary 偶然重合的有限样本扰动项。给出该扰动项在词汇大小 V 与 overlap 系数 ρ 下的衰减率，并实验验证 transformer 与 kernel theory 预测一致的 scaling。

## 核心结论
- Template task 上"模型靠抽象 vs 靠 token 名"的二分被改写为一个**连续谱**，由 overlap 项 ρ 控制。
- 在 V → ∞、ρ → 0 极限下，trained transformer 收敛到 ideal template classifier。
- 在 next-token prediction 设定下（必须"emit" unseen 符号）该清晰分解 **不再成立**——预测层强制注入 token-name 依赖。

## NTP 归类
- **类别**: NTP-mech (边界澄清)
- **理由**: 把"transformer 是否做抽象推理"这一旧争议从全有/全无变为可量化扰动；同时显式指出"NTP 的 readout 强制 token-name 依赖"是 fixed-label vs next-token 的本质差异。

## 与已有工作的关系
- 与 Anthropic / Olsson induction-head 工作互补：本文给出当样本量与词汇大小到达何种比例时 induction-head 应主导 vs 名称偏置应主导。
- 与今日 2605.10799 (CoT format confound) 同属"readout 效应主导了我们对 reasoning 机制的测量"主线。
- 对 NTP-mech 候选 C1 (depth horizon) 形成方法论补充：在 fixed-label 控制实验中可分离 architecture limit vs vocabulary confound。

## 是否提出新理论边界
yes (soft) — perturbation term 在 (V, ρ, m) 上的显式收敛率；并给出 NTP setting 下分解失效的 impossibility。

## Reviewer note (Xander)
工作的 scope 自承很窄（kernel regime + 固定标签），但提出的 "NTP readout 强制 token-name 依赖" 是被很多 emergent abstraction 论文忽视的盲点。结合今日 2605.10799 的 readout-dominance 证据，本周开始形成一个共同主题：**许多被解读为"NTP 推理机制"的现象其实是 readout / format 效应**。最强反对：fixed-label 与 NTP 的 gap 是否能被 prompt-format 缩小，本文没回答。
