# [arXiv:2605.23024] The Deterministic Horizon: Impossibility Results as Design Specifications for Trustworthy AI Systems

- **Authors / Org**: Dongxin Guo (单作者 thesis-style work)
- **Published / Updated**: 2026-05-21 / 2026-05-21
- **Primary cat / cats**: cs.AI / cs.AI, cs.CC, cs.CL, cs.LG
- **Link**: https://arxiv.org/abs/2605.23024

## 核心问题
是否存在一个**只由架构参数（层数、嵌入维度）决定**的推理深度上界，使得超过该上界后任何训练数据、loss、adapter rank 都无法显著恢复 accuracy？文章把若干 impossibility 结论从"理论好奇心"包装成可计算的部署前设计参数。

## 方法
作者构造一个 residual stream 上的 capacity 不变量，论证在 reasoning depth 超过某阈值后，accuracy 服从一个超指数下降的信息论上界；并在 12 个 transformer 架构上测量该阈值（被称为 Deterministic Horizon）落在 19–31 之间。补充结果：用电路复杂度 unconditional lower bound 证明 constant-depth prime-modulus circuits 无法做 modular exponentiation；以及把同一论证套到 preference learning、multi-stage retrieval、truthful auction、ZK 验证等子领域。

## 核心结论
- 给定 layer count 与 embedding width，存在**部署前可计算**的推理深度天花板 H*，训练再多数据也无法越过。
- 实测 12 个 transformer 架构 H* ∈ [19, 31]；在 optimal-length traces 上 fine-tune 仅能恢复 < 4 个百分点。
- modular exponentiation 在 constant-depth 模素数电路下有 unconditional lower bound（与已知 Furst–Saxe–Sipser 系列结果一致）。
- 跨子领域应用：misspecified preference model 的 sample complexity 出现不连续跳跃；多阶段 retrieval pipeline 至少需要 stage 数个独立 metric。

## NTP 归类
- **类别**: NTP-mech 候选 (架构 → 推理深度硬上界)
- **理由**: 若 capacity 不变量论证成立，则**只靠 NTP + scaling** 触不到超过 H* 的多步推理任务，这是一个机制级、不可通过 scale 移除的 bound。

## 与已有工作的关系
- 与 Merrill & Sabharwal (TC⁰ Transformers) 的电路复杂度上界互补：那一支证明 *表达能力* 受限，本文把它转成一个 *可测量的训练后剩余 error*。
- 与今日 2605.19944 (Measure-Theoretic Analysis) 高度共振：后者也得到 Dyck-k → TC⁰ 严格 depth 下界，两文形成"架构 depth = 推理 depth 上界"的双重证据。
- 与今日 2605.18079 形成对照：后者证明 low-precision softmax + CoT 仍可模拟 TM——即推理 depth 可以通过 CoT 摊到 token 维度上，对本文 "depth 即天花板" 是温和的反例。

## 是否提出新理论边界
yes — 给定 architecture (L 层, d 维)，存在 H*(L, d) 使任意 distribution、loss、adapter rank 下 reasoning depth > H* 的任务 accuracy 上界 super-exponentially decay。论文称其可在部署前计算，但具体闭式未在 abstract 给出。

## Reviewer note (Xander)
单作者、跨子领域射程过广，flagship 实测 H* ∈ [19,31] 的测量协议是否会被 CoT / scratchpad / 工具调用绕开，是最强反论。论证形式若严格，则是 NTP-mech 候选清单第一档；但需对照 2605.18079 的 CoT-扩展 expressivity 重新检视——"depth horizon" 是否只在 **no-CoT** 设定下成立。归档为候选，标注"待复现"。
