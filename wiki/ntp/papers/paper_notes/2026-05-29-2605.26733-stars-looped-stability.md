# [arXiv:2605.26733] Stabilizing Recurrent Dynamics for Test-Time Scalable Latent Reasoning in Looped Language Models

- **Authors / Org**: Xiao-Wen Yang, Ziyu Han, Xi-Hua Zhang, Wen-Da Wei, Jie-Jing Shao, Lan-Zhe Guo, Yu-Feng Li
- **Published / Updated**: 2026-05-26
- **Primary cat / cats**: cs.LG (cs.AI)
- **Link**: https://arxiv.org/abs/2605.26733

## 核心问题
Looped LM (depth recurrence) 在 test-time 增加 iteration 应当带来 *免费* 的 latent-reasoning 提升，实际却在某一深度后 *崩塌*。这是 latent reasoning 的根本不稳定，还是训练目标-架构不匹配？

## 方法
- 在 LoopLM 的 latent dynamics 上做谱分析，量化 stability (Jacobian spectral radius) 与 effectiveness (reasoning loss) 的 trade-off；
- 把 reasoning 重新概念化为 *uncertainty reduction*，要求 latent 轨迹收敛到 *asymptotically stable fixed point*；
- 提出 STARS = STAbility-driven Recurrent Scaling：训练时对随机 loop 深度采样、用 **Jacobian Spectral Radius 正则** 约束 latent map 谱半径；
- 算术任务 + 复杂数学 reasoning benchmark 上扫 iteration depth。

## 核心结论
- 现有 LoopLM 在 4–16 步后系统性退化，谱半径 >1；
- STARS 在 *训练成本不变* 下把 spectral radius 压到 <1，test-time 加深 → 性能单调或饱和而非崩塌；
- 数学 reasoning 上同时提升 peak 性能且消除深度崩塌。

## NTP 归类
- **类别**: counter-evidence (对 "NTP 架构下 latent-only reasoning 必然不稳定" 的弱主张)。
- **理由**: 把 latent-reasoning 的不稳定性归因到 *训练目标缺少稳定性约束* 而非 NTP 范式本身；这是 objective-engineering 路线在 *深度* 维度上的一次成功补丁。

## 与已有工作的关系
- 与今日 [2605.26379] LeJEPA-identifiability 并列：两者都把"latent 空间该满足何种结构性约束"作为核心 design lever；本文是 *动力学侧* (Jacobian radius)，LeJEPA 是 *分布侧* (Gaussian)。
- 与 N2 (TC⁰ wall) 直接相关：TC⁰ wall 的 *经典* 破墙手段是 CoT (token-space recurrence)，本文是 *latent-space recurrence*；如果 STARS 稳定，CoT 与 latent loop 在 test-time scaling 上构成 *双轨* 而非 *互斥*。
- 与 [2605.24956] NITP (auxiliary loss 工程) / [2605.05438] Semantic-Loss / [2605.11467] ProFIL 一起，把 objective-engineering 路线扩到 **第四个工程 lever**：Jacobian 正则。
- 削弱 §10 候选 C1 (deterministic horizon) 在 *latent 深度可变* 模型下的紧致性——C1 假定 forward-pass capacity 固定，STARS 类模型把 forward-pass 变成 *自适应深度*，C1 必须重新参数化为 (L, d, *converged loop depth*).

## 是否提出新理论边界
no formal — 但隐含 *设计原则*: "test-time scalable latent reasoning ⟺ trained-stable latent map (spectral radius <1)"。

## Reviewer note (Xander)
最干净的部分是 *机制 → 修复* 链路：找到不稳定的谱学根源、提出对应正则、验证 trade-off 缓解。最弱的是 *任务 transfer*：算术 + 数学 reasoning 跑通不代表自然语言长程 reasoning 同样稳定 (Jacobian 假设在 discrete-token + 长 context 下未必良定义)。在 survey §10 候选 C1 的修订表里应补注："C1 的 forward-pass 参数对 *fixed-depth* NTP 严格，对 *trained-stable looped* NTP 需替换为 *converged depth*"。
