# [arXiv:2605.19944] A Measure-Theoretic Analysis of Reasoning: Structural Generalization and Approximation Limits

- **Authors / Org**: Yuyang Zhang, Yifu Zhang, Xuehai Zhou, Xiaoyin Chen
- **Published / Updated**: 2026-05-19 / 2026-05-19
- **Primary cat / cats**: cs.LG / cs.LG, cs.AI, cs.CC, cs.CL
- **Link**: https://arxiv.org/abs/2605.19944

## 核心问题
LLM reasoning 的 OOD 泛化为什么会塌？能否用 optimal transport 把"reasoning 轨迹分布漂移"形式化，从而给出**架构层面的可证 generalization bound**？

## 方法
把离散推理轨迹嵌入连续 metric space，用 Wasserstein-1 距离量化 domain shift；通过 Kantorovich 对偶把 OOD risk 上界拆成两项：(i) 架构 Lipschitz 常数，(ii) functional approximation limit。然后做两件事：(a) 比较 Absolute Positional Encoding vs Rotary Embeddings 的 shift invariance；(b) 把"sequential backtracking"映射到 Dyck-k 语言，套 TC⁰ 电路 depth 下界。

## 核心结论
- APE 不保持 shift invariance → Lipschitz = Ω(1)，期望 risk 不可压缩；RoPE 保持 equivariance 故可被 bound 住。
- Dyck-k 映射意味着 TC⁰ transformer 存在严格 depth 下界；**scaling width 无法替代 depth**（Barron 空间近似下界使然）。
- 54 个 transformer 配置在组合搜索任务上的 generalization risk 与 Wasserstein domain shift 单调相关，验证理论。

## NTP 归类
- **类别**: NTP-mech 候选 (depth vs. width 不对称 + position encoding 选择 → 推理上界)
- **理由**: 给了一个在固定架构下 NTP 训练**无法越过**的 depth lower bound，且证明用 width scaling 也救不回来。

## 与已有工作的关系
- 与 2605.23024 (Deterministic Horizon) 形成互证：两者从不同形式化得出"architecture-bound 推理 depth"结论。
- 与 Hahn 2020 / Merrill & Sabharwal TC⁰ 系列直接续航；新颖点是把 OT/Wasserstein 引入 reasoning 泛化分析。
- APE vs RoPE 的论断与 Press 等 ALiBi 工作的"长度外推"经验观察一致。

## 是否提出新理论边界
yes — Risk_OOD ≤ Lipschitz(arch) · W₁(P_train, P_test) + Approx(arch)，且其中 Dyck-k 推理任务在 TC⁰ depth d 下 Approx ≥ const > 0。

## Reviewer note (Xander)
对 APE/RoPE 二分太干净，需要警惕：实际 RoPE 在长 context 也并非真的 shift-invariant (有效角度饱和)。Dyck-k → TC⁰ 这一支是经典结论，论文的贡献在于把它装进一个 generalization risk 不等式里。需要看正文 54 配置的 confound 控制（参数量、训练 token、optimizer），否则"理论 vs 实验"的 monotone 关系容易被 over-fitting 解释。
