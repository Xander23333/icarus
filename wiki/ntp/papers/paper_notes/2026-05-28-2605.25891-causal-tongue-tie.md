# [arXiv:2605.25891] Causal Tongue-Tie: LLMs Can Encode Causal Direction, But Their Yes/No Outputs Fail to Express

- **Authors / Org**: anonymous (pre-print)
- **Published / Updated**: 2026-05-26
- **Primary cat / cats**: cs.CL
- **Link**: https://arxiv.org/abs/2605.25891

## 核心问题
LLM 在 anti-commonsense causal 题目上的低 Yes/No 准确率，是真的"无法做 causal reasoning"，还是仅仅是 verbal interface 与内部表征解耦？

## 方法
在 CLadder anti-commonsense 子集上：固定一个**线性 probe** 从模型 hidden state 解码 evidence-supported 答案；同时记录模型的 Yes/No 输出；对照二者准确率差，提出"Causal Tongue-Tie" gap 概念，把 wrong Yes/No 分解为两种独立失败：(i) 内部无信号；(ii) 内部有信号但 verbal interface 无法表达。

## 核心结论
- Probe 准确率 ≈ 0.97；Yes/No 准确率 ≈ 0.5。
- ≈ +0.5 gap 几乎全部来自 (ii)：内部正确、口头错。
- output-only causal benchmark 的 *"对"和"错"都不能直读为* mechanism 结论：对 ≠ 理解，错 ≠ 不能。

## NTP 归类
- **类别**: counter-evidence（对"LLM 无 causal reasoning 能力"的削弱） + NTP-mech 边界澄清（NTP readout 是 systematic bias 注入点）
- **理由**: 把外显 yes/no 失败的来源从"机制级缺失"显式拆出一项"readout-side 表达失败"。这一项被证明可由极轻 (linear probe) 干预绕过——属 readout 工程层。

## 是否提出新理论边界
no formal bound — 但给出 *可证伪的方法论命题*：任何基于 yes/no accuracy 的"LLM 缺 causal X"声称，必须附带 probe-vs-output 双轨证据，否则结论不可读。

## Reviewer note (Xander)
最强反论：linear probe 的"高准确率"可能不是 *causal direction* 表征，而是任务 surface feature（CLadder 题型规律）被线性可分。但作者在 anti-commonsense 子集做的设计部分屏蔽了这点（commonsense bias 与 evidence 方向相反时 probe 仍准）。强度合格；够进入 "readout-side 主导" 弹药库。
