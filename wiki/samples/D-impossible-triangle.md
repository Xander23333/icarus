# 第十一章　Benchmark 的不可能三角

## 一、三角的诅咒

二〇二六年春天，一位 frontier 模型公司 agentic coding 评测的负责人，在一次内部分享会上画了一个三角形。三角形的三个顶点分别写着：**反作弊**（uncontaminable）、**可比**（comparable）、**真实**（ecologically valid）。他在三角的中心打了一个大大的红叉，然后说了一句后来在公司内部流传甚广的话：「这三样东西，你只能选两样。第三样，会在你睡觉的时候被人拿走。」

这并不是一句俏皮话。它是过去六年人工智能评测史的浓缩。从二〇二〇年的 MMLU，到二〇二一年的 HumanEval，到二〇二二年的 GSM8K 与 MATH，到二〇二三年的 SWE-bench，再到二〇二四年的 HLE（Humanity's Last Exam）和 LMArena——每一次新基准的诞生，都是对前一个基准死亡的悼词。而每一次死亡的死因，都可以被归纳到那个三角形的某一条边上。

如果一个 benchmark 足够**真实**（贴近真实任务分布），它就一定可以被检索、被模仿、被污染——因为真实世界的语料终将进入下一次预训练的爬虫管道。如果它足够**可比**（题目固定、评分确定），它就一定会被过拟合——因为固定的题目意味着可被记忆的答案。如果它足够**反作弊**（题目私有、动态生成），那么不同实验室、不同时间、不同版本之间就失去了同台竞技的基础——你无法比较两个从未在同一张试卷上答题的考生。

这便是 benchmark 的不可能三角。它不是一个工程难题，而是一个结构性悖论。理解了它，就理解了为什么人工智能的评测史，看起来如此像一部西西弗斯式的循环剧。

## 二、三个尸体：MMLU、HumanEval、GSM8K

让我们先去三具尸体旁边走一走。

**第一具尸体是 MMLU。** 二〇二〇年九月，加州大学伯克利分校的 Dan Hendrycks 与合作者发布了 *Measuring Massive Multitask Language Understanding*。一万五千多道题，五十七个学科，从高中物理到法学院考试。它在二〇二〇至二〇二三年间几乎是「通用智能」的代名词。GPT-3 拿到 43.9%，GPT-4 跃升到 86.4%，Gemini Ultra 宣布超越人类专家。然后呢？然后人们开始发现，MMLU 的题目大量来自公开的备考网站、Quizlet 卡片、Wikipedia 的延伸阅读。Jacob Steinhardt 团队二〇二三年的一篇分析显示，相当比例的题目可以在 Common Crawl 中以接近原文的形式找到。MMLU 不是被破解的，它是被**吸收**的——像一块糖被慢慢化进了预训练的咖啡里。Hendrycks 本人后来承认，MMLU 已经「饱和」，并转向了更难的 HLE 项目。

**第二具尸体是 HumanEval。** 二〇二一年 OpenAI 发布的 164 道 Python 函数补全题，曾经是代码生成模型的金标准。Codex 拿到 28.8%，被认为是革命性的。到二〇二四年，几乎所有主流模型都能拿到 90% 以上。Sébastien Bubeck 在 *Sparks of AGI* 中把它当作 GPT-4 编程能力的证据，但短短一年之后，HumanEval 已经成为新闻稿里的脚注，而不是评测的脊梁。原因和 MMLU 一样：164 道题太少，公开太久，它们的标准答案与变体早已散落在 GitHub、Stack Overflow、博客文章里。一个模型在 HumanEval 上的高分，到了二〇二四年，只能证明它「读过书」，不能证明它「会写代码」。

**第三具尸体是 GSM8K。** Karl Cobbe 等人二〇二一年发布的八千多道小学数学应用题。它的命运最具戏剧性：二〇二三年，Scale AI 的研究者用人工重写了 GSM8K 的题目（保持难度与结构，但更换数字与措辞），发布了 GSM1K。结果令人不安——多个高分模型在 GSM1K 上的表现比 GSM8K 下降了 8 到 13 个百分点。差距最大的几个模型，恰恰是那些以「数学推理」为卖点的模型。GSM8K 没有被「学会」，它被「背下来」了。

三具尸体，三种死法，但死因都写在同一张诊断书上：**真实 + 可比 = 不可反作弊**。

## 三、塑造者的进退两难

如果说前面是受害者的故事，那么这一节是设计者的故事——那些试图在三角形里走钢丝的人。

**Dan Hendrycks** 在 MMLU 饱和之后，主导了 HLE（Humanity's Last Exam）。这个名字本身就是一个隐喻——「人类的最后一场考试」，仿佛在承认每一次评测都注定走向死亡，只是想再撑久一点。HLE 的策略是「极端难度 + 严格私有 + 专家众包」，试图通过题目的稀缺性来抵抗污染。但代价是：题目数量不足以覆盖能力分布的细粒度，评分需要专家介入因而**可比性**受损。[推测] Hendrycks 本人在播客中也暗示过，HLE 的寿命可能只有 18 到 24 个月。

**Nicholas Carlini**——前 Google Brain、现 Anthropic 的安全研究员——走了另一条路。他长期主张「benchmark 应该是动态的、对抗性的、不可重复的」。他在二〇二三年的一篇博客中写道：「任何一个静态 benchmark，从它发布的那一刻起，就开始死亡。」Carlini 的方案接近三角的**反作弊**顶点，但他坦承，这条路的代价是放弃 leaderboard 文化——而放弃 leaderboard，意味着放弃了行业沟通的通用货币。

**普林斯顿的 SWE-bench 团队**（Carlos Jimenez、Ofir Press 等）则选择了第三条路：用真实的 GitHub issue 与 PR 构造任务，并配备可执行的测试套件作为评分。SWE-bench 一度被誉为「agentic coding 的 ImageNet 时刻」。但它很快遇到了三角的另一条边——**可比性**：不同 agent scaffold（AutoCodeRover、SWE-agent、OpenHands）的差异远大于底层模型的差异，导致同一个 GPT-4o 在不同框架里的得分能相差 20 个百分点。SWE-bench Verified（由 OpenAI 与普林斯顿合作筛选）试图缓解这个问题，但又因数据集变小而牺牲了统计可靠性。

**LMArena**（原 Chatbot Arena）则代表了第四种妥协：彻底放弃静态题目，改为人类盲评配对。它在**真实**与**反作弊**两个顶点上都取得了胜利——题目由用户实时输入，无法预先污染。但代价是**可比性**的形变：Elo 分数依赖用户偏好，而用户偏好系统性地偏爱**更长、更礼貌、更格式化**的回答。Anthropic 的研究者发现，仅仅让模型回答更长，就能在 Arena 上提升 50 到 100 分。Arena 没有作弊，但它测量的「智能」与人们以为的「智能」之间，存在一道悄悄滑动的鸿沟。

每一个塑造者都站在三角形的一条边上，背对一个深渊。

## 四、作弊者的视角：Goodhart 是常态，不是异常

我们习惯把数据污染（contamination）和 Goodhart's Law 描述为评测的「失败模式」。但如果我们换一个视角——作弊者的视角——会发现，污染与过拟合并不是系统的 bug，而是系统的**默认行为**。

考虑一个二〇二五年的中等规模实验室。它的目标函数是：在下一次发布会前，让旗舰模型在公开 benchmark 上超越对手 2 个百分点。它的训练数据来自 Common Crawl + 私有合成数据 + RLHF。请问，它有任何**结构性**的动机去主动剔除 MMLU、GSM8K、HumanEval 的题目吗？答案是：几乎没有。

第一，剔除是昂贵的——精确去重需要题目级的指纹匹配，而许多 benchmark 题目是从更早的语料中改编而来，根本无法干净剔除。第二，剔除是危险的——一旦剔除导致分数下降，会被解读为「模型变弱了」。第三，更隐蔽的是：哪怕实验室真心想剔除，**合成数据**这一环节会把 benchmark 的分布特征以另一种形式重新引入——因为生成合成数据所用的「教师模型」本身就在那些 benchmark 上训练过。

这就是 Goodhart's Law 的现代化身：「当一个指标变成目标，它就不再是一个好的指标。」Charles Goodhart 在一九七五年说这句话时，谈的是英国货币政策。半个世纪后，它成了 AI 评测的统治性定律。每一个 benchmark 一旦进入 leaderboard，它就从「测量工具」变成了「优化目标」，而被优化的目标，必然失去测量的能力。

更悲哀的是，这个过程**不需要恶意**。没有人需要主动作弊，整个梯度下降的机制就会自动完成作弊。模型工程师只需要诚实地训练、诚实地评测、诚实地发布——而 benchmark 会在他们不知情的情况下死去。

## 五、二〇二六：评测者的绝望与共鸣

回到本章开头的那个会议室。那位负责人画完三角形之后，做了一件更让团队震动的事——他在三角形旁边写下了四个 benchmark 的名字：SWE-bench Verified、LiveCodeBench、TerminalBench、自研的 QwenCode-Hold-out，并在每一个名字旁边注上了「预计死亡时间」。最近的一个，是九个月。最远的一个，是十八个月。

「我们不是在建评测，」他说，「我们是在建一座座墓碑，并提前刻好日期。」

这种绝望不是任何一家实验室独有的。OpenAI 的 evals 团队、Anthropic 的 model evaluation 组、DeepMind 的 capability eval 部门，都在二〇二五至二〇二六年间经历了同一种危机——**评测的半衰期，正在变得比模型迭代周期还短**。一个新 benchmark 从发布到饱和，曾经需要三年（MMLU），后来变成一年（HumanEval），现在只需要六个月（SWE-bench Verified）。在某些子领域（如数学奥赛、竞赛编程），半衰期已经短到了**单次模型发布周期内**——这意味着一个 benchmark 在它被首次报告分数的同一份技术报告里，就已经死亡。

他们团队的应对策略是「滚动式 hold-out + 私有验证集 + 多 scaffold 平均」，但他自己承认，这只是把死亡推后，不是阻止死亡。[uncertain] 他在那次分享的最后说了一句话，我无法确认原话，但大意是：「也许我们这一代评测人的工作，不是找到那个永恒的尺子，而是接受尺子会不断融化这件事——并学会在融化中继续测量。」

这是一种 Harari 式的觉悟。在《人类简史》里，Harari 反复指出，人类历史上许多看似永恒的制度（货币、宗教、国家），其实是脆弱的「想象的共同体」，靠集体相信而存在。Benchmark 也是这样一种想象的共同体——它的有效性，不来自数学，而来自**所有人同意暂时相信它**。一旦相信瓦解，benchmark 就死了，无论它的题目还在不在。

那个三角形之所以是不可能的，不是因为工程上做不到，而是因为它要求人类同时相信三件互相否定的事情：相信测量是客观的、相信比较是公平的、相信现实是可被还原的。我们能同时相信其中两件，从来没能同时相信三件。

而二〇二六年的这位评测负责人，和他在伯克利、在普林斯顿、在伦敦国王十字的同行们，正站在第三次相信即将瓦解的门槛上。门后面是什么，没人知道。但门前的那个三角形，仍然画在白板上，红叉仍然鲜艳。
