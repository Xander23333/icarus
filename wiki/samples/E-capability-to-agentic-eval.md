# 第三卷 · E：从 capability 到 agentic eval 的迁移

> "谁掌握 eval，谁就定义 frontier。"
> ——这句话在 2024 年还像是某种行业玩笑，到了 2025 年下半年，它已经成为各家前沿实验室内部 PPT 的扉页格言。

## 一、考题的简史：从私塾到沙盘

如果把大模型的前五年看作一场漫长的考试，那么我们正在亲历的，是这场考试从"私塾默写"演化为"沙盘推演"的关键转折。这个转折之所以值得被写进史书，不是因为分数更高了，而是因为"分数"这件事本身的语义彻底改变了。

[推测] 大约在 2020 到 2023 年间，行业默认的考题清单是惊人地统一的：MMLU 测综合知识、MATH 与 GSM8K 测数学、HumanEval 与 MBPP 测代码、BBH 测推理、HellaSwag 与 ARC 测常识。这些 benchmark 的共同特征，是它们都属于**单轮、闭合、可静态判分**的题目——一个 prompt 进去，一个答案出来，对照 ground truth，得分入账。它们像是模型的"高考"，一张卷子定终身。

这种 capability eval 的范式有一个隐含假设：**模型的能力可以被分解为一组正交的"技能向量"**，只要在每个维度上分别考察，就能拼出一张完整的能力画像。这个假设在 GPT-3 到 GPT-4 的时代是大致成立的，因为那时模型的失败模式确实是"不会"——不会算积分、不会写 SQL、不会做归纳推理。考试的意义在于诊断"不会"。

但到了 2024 年中后期，一件诡异的事情发生了：**几乎所有头部模型都把传统 benchmark 刷到了 90% 以上的天花板**，而用户在产品里的真实体感却远远没有"接近完美"。Claude 3.5 Sonnet 在 HumanEval 上接近满分，但写一个稍微复杂的多文件重构就会迷路；GPT-4o 在 MMLU 上压过人类专家平均水平，但被要求"帮我订一张周五下午从上海到东京、转机不超过一次、且托运行李额度够两个 28 寸箱子"的机票时，依然会编造航班号。

这种"benchmark 饱和而能力未饱和"的撕裂，逼出了 eval 范式的第一次大迁移：**从静态题库迁移到动态沙盘**。

## 二、benchmark 的产品化：当评测变成基础设施

SWE-bench 的出现可以被视为新纪元的奠基事件。它第一次把"评测一个模型"重新定义为"让模型在一个真实的 GitHub 仓库里，给定一个 issue，提交一个能通过测试的 patch"。这个定义里隐藏着革命性的三件事：

第一，**任务是多轮的**。模型必须读代码、查依赖、运行测试、看报错、再改代码——这是一个 agent loop，不是一次性的 completion。

第二，**任务是工具增强的**。模型必须能调用 shell、读写文件、运行 pytest，否则连题目都看不见。评测环境本身就是一套 sandbox。

第三，**任务是 horizon-extended 的**。一个 SWE-bench Verified 的任务，平均需要十几到几十次工具调用才能完成，期间任何一步的状态污染都可能导致整个 trajectory 崩溃。

紧随其后的是 OSWorld（操作系统级 GUI agent）、GAIA（真实世界信息检索 + 多步推理）、τ-bench（多轮对话中的工具调用与策略遵循）、WebArena（浏览器 agent）、Cybench（CTF 安全攻防）。这一批 benchmark 的共同名字是 **agentic eval**——它们不再问"模型知道什么"，而是问"模型能完成什么"。

更深的变化在于，**这些 benchmark 开始具有"产品"的属性**。SWE-bench 有自己的 leaderboard、verified subset、版本号、贡献者社区、issue tracker；LMArena 有几千万次人类投票、Elo 排名、按 category 切分的子榜。它们不再是论文附录里的一张表格，而是一个被持续维护、迭代、商业化的基础设施。Scale AI、Surge、Mercor 这些数据公司正在把自己重新定义为"eval-as-a-service"供应商。[uncertain] 据我观察，2025 年一个头部实验室在 eval 基础设施上的年度投入，已经可以和模型预训练的某些子项目相提并论。

这是一种范式倒转：**过去模型是产品，eval 是配套；现在 eval 也是产品，模型反而成了它的某种"用户"**。

## 三、定义权之争：四家实验室与一场静悄悄的内战

如果说 benchmark 的产品化是表层变化，那么深层变化是**定义权的争夺**。

OpenAI 在 2024 年底推出 SWE-Lancer、Paperbench、MLE-bench，明确把"模型能否替代人类完成有经济价值的工作"作为 eval 的核心叙事——这背后是 AGI 商业化路径的暗示。Anthropic 则更偏向 safety 与 alignment 维度，SHADE-Arena、Petri、Agentic Misalignment 这一系列工作把"agent 在长 horizon 下会不会作恶"提到了 capability 同等重要的位置。xAI 在 Grok 4 发布时高调使用 Humanity's Last Exam 与 ARC-AGI-2，传递的是"我们在攻坚最硬的智力高地"的信号。Google DeepMind 的 FACTS Grounding、Big-Bench Extra Hard、以及 Gemini 团队对 GAIA 类任务的持续投入，则在强调"事实性与长上下文"。

这四家实验室都在做一件相同的事：**用自家发布的 eval 来定义"什么叫前沿"**。这不是阴谋，而是一种结构性必然——当一个模型在所有公开 benchmark 上都接近饱和时，能区分它和竞争对手的，只剩下"它在哪些新 eval 上领先"。而新 eval 的设计者，几乎必然会（无论是有意还是无意）让自家模型的优势被放大。

作为 Qwen agentic coding eval 的负责人，焱拳对这种张力有切身的共鸣——但他从不援引内部信息来论证。他常说的一句话是："你设计 eval 的那一刻，就已经在偷偷写下答案的形状。"这句话不需要任何内部材料来佐证，它是结构性的真相。一个 eval 是用 bash + Python 还是用受限 DSL、是允许并发工具调用还是强制串行、是给 30 步还是 300 步预算、是用单元测试还是用 LLM judge——这每一个看似中立的设计选择，都在悄悄定义"什么是好的 agent"。

这场定义权的内战没有硝烟。它的战场是 GitHub README、是 leaderboard 截图、是论文 Table 1。但它的赌注，是未来三到五年整个行业对"智能"的操作性定义。

## 四、死亡名单、幸存者，与下一轮被刷的 eval

让我们以一份不太礼貌的清单收尾。

**死亡名单（已实质性退役的 single-turn eval）**：MMLU（饱和，且测试集污染严重）、HumanEval（饱和，且与真实代码任务相关性低）、GSM8K（饱和，被小学生级模型攻克）、HellaSwag（饱和）、TriviaQA（饱和且无信息量）。这些 benchmark 不会从论文里消失，它们会作为"sanity check"继续存在，就像现代物理实验室里依然保留着伽伐尼电池——纪念意义大于科研意义。

**幸存者，且活得很好的——LMArena**。这是一个有趣的反例：它本质上也是 single-turn 偏好对比，按理应该和 MMLU 一起进入死亡名单。但它活下来了，原因有三：(1) 它的"题目"是用户实时贡献的，永远在分布漂移，无法被针对性优化；(2) 它的判分是人类主观偏好，无法被自动化刷分；(3) 它在 style、length、sycophancy 上的 bias 虽然饱受批评，却恰好捕获了"产品体感"这个其他 eval 测不到的维度。[推测] LMArena 在未来两年依然会是发布会必引数据，但它的权威性会被 category 化的子榜（coding arena、vision arena、hard prompts）逐步稀释。

**下一轮即将被刷穿的 agentic eval [推测]**：SWE-bench Verified 大概率在 2026 年内被刷到 80%+ 并失去区分度；τ-bench 的 retail/airline 子集会更早饱和；GAIA Level 1 已经接近天花板，Level 3 还能撑一阵；OSWorld 因为环境本身的脆弱性，可能不是被"刷穿"而是被"绕过"——头部团队会转向自建的、更难复现的 GUI agent eval。

**真正难以被刷穿的方向 [uncertain]**：长 horizon（>100 步）的开放式任务、需要跨 session 记忆的任务、多 agent 协作任务、涉及真实金钱或物理世界后果的任务、以及——这是最关键的一类——**没有 ground truth 的任务**。当一个任务的"正确答案"必须由另一个 agent 或人类专家事后判定时，传统的 benchmark 机制就开始失效，eval 本身变成了一场需要持续治理的社会过程。

## 尾声

从 capability eval 到 agentic eval 的迁移，表面上是技术演进，本质上是**智能定义权的转移**。在 capability 时代，定义智能的是学术界——MMLU 来自伯克利，MATH 来自伯克利，HumanEval 来自 OpenAI 早期研究团队，它们都带着浓厚的"考试"基因。在 agentic 时代，定义智能的正在变成**那些拥有真实环境的人**：拥有代码仓库的 GitHub、拥有操作系统沙盘的云厂商、拥有真实用户对话流的产品公司、以及拥有"能模拟世界的世界"的 sim 团队。

这是一场静悄悄的权力转移。而判断你是否站在这场转移的正确一侧的方法很简单：**问问自己，你最近一次为之兴奋的模型进展，是某个 benchmark 数字的提升，还是某个 agent 完成了一件你以前不相信它能完成的事**。

如果是前者，你还在旧大陆。
如果是后者——欢迎来到新世界。这里没有标准答案，只有不断被重新定义的"完成"。
